8 // This class is the concrete implementation of the
9 // cuda_concrete_domain considering list of bitmaps
10 // {min, max} of (non) contiguous domain's elements.
11 //
12 // @note A bitmap list is represented as follows:
13 //
14 // ...|| L | U | BITMAP | L | U | BITMAP |... ,
15 //
16 // where L/U is the lower/upper bound of the elements of
17 // BITMAP when the BITMAP has been created/added to the list.
18 // BITMAP has a fixed size (i.e., fixed number of chunks)
19 // correspoinding to the number of chunks needed to represent
20 // the largest set of consecutive values (i.e., max ( U - L + 1 )).
21 //
22 // @note This class does not use an actual list data structure.
23 // Instead, it uses a domain bitmap representation.
24 // This is done in order to write C code that could be used
25 // later on CUDA kernels.
26 //
27 // @note This class do not use any auxiliary member to handle the
28 // methods that will modify the iternal domain's representation
29 // (i.e., pointers to each bitmap). This is done for
30 // implementation reasons in order to "directly replay" the same
31 // algoithms in C CUDA code.
32 //
