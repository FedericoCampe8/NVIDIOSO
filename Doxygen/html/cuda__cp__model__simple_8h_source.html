8 // This class represents a concrete CP model on CUDA.
9 // Its behaviour is the same as CudaCPModel (see "cuda_cp_model.h")
10 // but it changes the way states are uploaded and downloaded on device.
11 // Uploads and downloads are performed according to the representation of domains
12 // used on CUDA, i.e., Boolean domains on Host are represented with 2 integers on Device
13 // while standards domains on Host are represented either with a pair of bounds or
14 // with a bitmap.
15 // Therefore this concrete class implements the above mapping.
16 // Note that events on domains are set again before states are copied to device
17 // (normally there are no events on domains before constraint propagation).
18 // This is done to allow device to refer directly to the event on a given domain
19 // without the need of retrieving it from the current elements in the domain and,
20 // therefore, reduce memory accesses on device.
21 //
